What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. CI means that every time a developer pushes code, automated checks run to make sure nothing is broken. CD means the code is automatically deployed to the live server after passing those checks. This process saves time, reduces human errors, and ensures the website is always up to date. In cloud systems, CI/CD is essential for fast and reliable software delivery.

My Pipeline

When I push code to the main branch, GitHub Actions automatically runs my workflow. It checks that all required files exist, then uploads the site and deploys it to GitHub Pages. Within seconds, my live website is updated without any manual steps.
